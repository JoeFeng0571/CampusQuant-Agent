一、项目背景与痛点（0:15 - 1:20）
【画面：校园场景，学生刷手机；切入问卷数据可视化动效，数字逐条浮现】
大家好，我们是 CampusQuant 团队。在开发这套平台之前，我们对大学生股票投资现状进行了问卷调查与深度访谈。

【画面：第一组数据卡片出现——"43.08% 通过社交媒体学习金融知识"】
超过 43% 的受访同学，金融知识的主要来源是短视频和社交媒体。 但这条路走得并不顺畅——78.46% 的人误判了一道基础题：一只股票下跌 5% 后再上涨 5%，并不是回本，而是略微亏损。这道题错误率近八成，折射出碎片化学习渠道带来的知识缺口。

【画面：第二组数据卡片——"超 70% 风险把控能力一般或完全不懂"】
超过 70% 的受访者坦言，自己对风险的把控能力一般甚至完全不懂。 在从未接触过投资的同学中，这一比例更高达 51.52%。知道要分散投资的人有六成，但真正付诸行动、建立起投资体系的，寥寥无几——这是一种典型的"知识与行为断层"。

【画面：访谈片段，用户画像"小林"出场——大二理工科学生，本金 300-500 元，设置了 15% 止损线】
我们的调研中，有一位典型用户叫小林。本金只有几百元，全部来自生活费结余。他有止损意识，清楚自己是"韭菜"，也知道差距在于"认知"。
他用了一句话概括自己的现状："我愿意花时间学，但不知道从哪里学起，也不知道自己学的对不对。"
这句话，几乎道出了我们目标用户的共同处境。

【画面：第三组数据卡片——"量化教学 64.62% / 自动风控 63.08% / 辅助决策超 50%"】
当我们问到"你最希望智能工具提供什么"，量化教学（64.62%）和自动风控（63.08%）并列为最核心的需求，超过半数用户偏好"辅助决策"模式——不是让 AI 替我操盘，而是让 AI 陪我思考，展示分析过程，帮助我成长。

【画面：切入平台登录页 auth.html，Logo 渐变出现】
这正是我们设计 CampusQuant-Agent 的出发点：不是又一款给结论的股票 App，而是一个让分析过程可见、让风险意识落地、让每一次模拟操作都成为真实学习的智能平台。

二、平台整体架构一览（1:20 - 2:10）
【画面：控制台导航栏，依次高亮各模块入口：控制台 / 模拟演练 / 个股分析 / 市场快讯 / 投教社区】
平台整体分为五大功能模块，导航栏清晰呈现，用户可随时在各模块之间自由切换。
技术层面，系统采用五层架构：Streamlit 前端、FastAPI 服务层、LangGraph 多智能体编排层、多源数据工具层与 SQLite 持久化层。
【画面：分析流程图动态展示：数据采集 → 四路并行分析 → 综合决策 → 辩论 → 风控 → 交易输出】
整个分析链路可以概括为：数据采集 → 四路并行分析 → 综合决策 → 冲突辩论（按需触发）→ 风险审查 → 模拟交易输出。
接下来我们逐一演示各核心功能。

三、功能模块演示

模块一：个股智能分析（2:10 - 4:20）
【画面：进入 analysis.html，左侧控制面板特写：三个市场选项卡"A股 / 港股 / 美股"横排，"A股"高亮激活】
这是平台最核心的功能——个股多智能体协同分析。
平台支持 A 股、港股、美股三大市场，用户可点击切换，系统将自动路由到对应的数据源。
【画面：在股票代码输入框填入"600519.SH"，点击"开始分析"按钮，按钮蓝青渐变色高亮】
我们以贵州茅台为例。输入代码，点击分析——系统随即启动 LangGraph 状态机，将任务分发给多个专业智能体节点并行执行。
【画面：顶部状态栏亮绿点闪烁，显示"分析中"；节点进度区域 #node-progress 中，六枚绿色徽章依次点亮：数据 / 基本面 / 技术面 / 舆情 / 组合 / 风控】
可以看到，六个节点徽章依次点亮——这正是 SSE 实时推流的效果：数据节点率先完成行情拉取与技术指标计算，紧接着基本面、技术面、舆情节点并行运行，用户可以全程实时看到系统的分析进度，决策过程一目了然。
【画面：分析完成后，结果卡片 #result-card 出现，四项核心结论并排展示：建议"观望" / 置信度"68%" / 参考价"?1680" / 风险"中"，各项分别以暖黄、蓝色、白色字体高亮】
分析完成后，系统输出四项核心结论：投资建议、置信度、参考价格与风险等级，简洁直观。
【画面：切换到"深度研报"标签页，研报正文区域出现，标题蓝色高亮、正文多段落排版，列举关键因素】
点击"深度研报"，可查看由各节点汇总生成的完整分析报告，包含投资逻辑、关键因素、目标价区间与主要风险项。
【画面：切换到"财务图表"标签页，柱状图占位区域出现，蓝青渐变柱体排列，底部说明文字"正式环境 ECharts 渲染"】
"财务图表"页签展示营收、利润等关键财务趋势，正式环境接入真实数据后由 ECharts 动态渲染。
值得一提的是，所有分析结果均通过 Pydantic 结构化 Schema 约束输出格式，包含置信度、目标价、风险因素等标准字段，彻底避免大模型输出格式漂移，保障前后端接口稳定。

四、创新亮点总结（8:30 - 9:15）
【PPT：黑色背景上，六条亮点逐条以渐变文字飞入】
总结 CampusQuant-Agent 的六大核心创新：
第一，多智能体协同分析——六个专业节点并行运行，告别单模型黑箱，分析过程完全透明；
第二，LangGraph 状态机工程化编排——支持条件路由、冲突触发辩论、检查点恢复，将大模型转化为可追踪的结构化流程；
第三，Pydantic 结构化输出——置信度、目标价、风险等级等字段强制约束，系统接口稳定可靠；
第四，混合 RAG 知识增强——BM25 稀疏检索与向量语义召回融合，降低大模型"幻觉"风险；
第五，SSE 实时流式可视化——节点徽章逐步点亮，用户全程看见决策形成过程，不再是等待黑屏；
第六，分析、风控、交易、学习四位一体——从研报生成到持仓体检，从模拟交易到社区复盘，构成完整教育闭环。
