## 1. 认证测试

### 1.1 使用用户提供邮箱测试

测试邮箱：`2254670449@qq.com`

结果：
- 登录前置测试：`401 邮箱或密码错误`
- 注册测试：`400 该邮箱已被注册`
- 再次登录：`401 邮箱或密码错误`

数据库核验：
- 该邮箱已存在于本地库
- 对应用户：`xq`
- 用户状态：`is_active = 1`
- 创建时间：`2026-03-24 15:26:16`

结论：
- 当前系统不存在“待邮箱验证”状态。
- 该邮箱是已注册账号，不是“等待验证码激活”的中间态。
- 从用户视角，这会形成明显死路：邮箱已注册，但前端没有找回密码或重置入口。

状态：`失败`

## 4. 体验问题与改进建议

### P0

- 认证页未说明当前没有邮箱验证码、激活邮件和找回密码能力，容易让用户误以为“宝塔邮箱没配好”。
  改进：在注册/登录页明确说明当前认证能力边界。

- 已注册邮箱再次注册时，只返回“邮箱已被注册”，但没有引导用户回到登录，也没有找回密码路径。
  改进：自动切回登录页并预填邮箱，给出明确下一步提示。

### P1

- 认证页接口地址硬编码为固定 IP，不利于宝塔反向代理、域名部署和多环境切换。
  改进：优先走当前域名同源，保留本地 `127.0.0.1` 开发分支。

- 持仓体检响应慢时，用户只会看到长时间 loading，难以判断是正在分析还是卡死。
  改进：增加预计耗时、超时说明和重试建议。

### P2

- 全站仍存在部分占位链接和死链风险，影响验收完整性。
  改进：统一补齐真实跳转或改为不可点击状态。
