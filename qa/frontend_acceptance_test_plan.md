# 前端交付验收测试文档

日期：2026-03-30
项目：CampusQuant
视角：前端验收 / 用户路径 / 模拟交易闭环

## 1. 验收范围

页面范围：
- `auth.html` 登录/注册
- `dashboard.html` 控制台
- `trade.html` 三市场模拟交易
- `analysis.html` 个股分析
- `market.html` 市场快讯
- `platforms.html` 持仓体检
- `community.html` / `post_detail.html` 投教社区
- `home.html` / `resources.html` 学习资源
- `team.html` / `index.html`

功能范围：
- 注册、登录、登录态保持、退出
- A 股、港股、美股模拟交易
- 个股搜索、行情、K 线、最小手数校验
- 持仓、账户、成交记录
- 个股分析与市场数据展示
- 持仓体检
- 社区帖子、评论、点赞
- 财商导师对话
- 页面导航、空态、错误态、关键文案

## 2. 验收原则

- 以用户前端路径为主，不只看接口通不通。
- 市场相关功能按现有“不同市场不同数据源”逻辑验收，不修改该逻辑。
- 测试结果分为 `通过`、`失败`、`风险`、`待补测`。
- 若涉及体验问题，必须提出改进方案。

## 3. 关键测试场景

### 3.1 认证链路

用例：
- 新邮箱注册成功并自动登录
- 已注册邮箱再次注册时给出可执行引导
- 正确邮箱+密码登录成功
- 错误密码登录失败并提示明确
- 登录后本地 token / username / user_id 写入
- 已登录用户再次进入认证页时自动跳转
- 退出登录后清理本地登录态

重点关注：
- 是否真的启用了邮箱验证码/激活邮件
- 是否存在“邮箱已注册但无法找回密码”的死路
- 宝塔反向代理下认证页接口是否走同源

### 3.2 三市场模拟交易

用例：
- A 股：搜索 `600519.SH`，查询最小手数、现价、K 线，下单校验
- 港股：搜索 `00700.HK`，查询最小手数、现价、K 线，买入 100 股
- 美股：搜索 `AAPL`，查询最小手数、现价、K 线，买入 1 股
- 资金不足时阻止下单并提示
- 下单成功后账户资产、持仓、成交记录联动刷新
- 交易页未登录时成交记录区给出明确登录引导

重点关注：
- A/HK/US 三市场入口都能独立完成交易闭环
- 港股 lot size 与美股 1 股逻辑是否正确
- 前端是否能正确识别并展示不同币种资金

### 3.3 行情与分析

用例：
- 个股搜索联想
- 个股分析分析流触发
- 市场指数、情绪、热榜加载
- K 线接口返回为空时的空态
- 行情失败时错误提示可理解

### 3.4 持仓体检

用例：
- 输入合法持仓后发起体检
- 后端处理较慢时前端 loading 态是否明确
- 失败时错误提示是否说明原因

### 3.5 社区与学习

用例：
- 社区帖子列表加载
- 登录后发帖
- 帖子详情加载
- 评论、点赞
- 学习资源与导航可达性

## 4. 通过标准

- P0：认证可用；三市场交易闭环可跑通；关键页面无阻断性死链
- P1：错误提示清晰；未登录路径有明确引导；用户能理解当前能力边界
- P2：弱网络/外部数据波动时能给出合理降级或提示

## 5. 本轮执行策略

- 第一优先级：注册、登录、登录态
- 第二优先级：A/HK/US 三市场模拟交易
- 第三优先级：社区、体检、导师、市场快讯
- 第四优先级：导航、空态和文案体验
