## 课程简介

> 本项目横跨两门课程的核心知识域，将理论落地于"AI 辅助投资决策"这一真实工程场景

| 课程知识点 | 本项目对应实现 |
| --- | --- |
| 数据驱动的状态估计 | 实时行情 + 财报数据驱动四路 Agent 分析 |

| 反馈控制与重试机制 | 风控拒绝 → 组合 Agent 修正 → 再审，最多 2 轮 |

## 研究意义

### 为什么要做这个？

中国在校大学生 **2.9 亿**，其中有投资意愿但缺乏系统指导的比例持续上升

> 💬 "散户七亏二平一赚"——A 股长期规律

### 三大痛点催生研究价值

- **信息不对称**：机构有研究团队，个人投资者只能看大 V 微博
- **决策非理性**：追涨杀跌、情绪化操作，缺乏系统性框架
- **教育资源碎片化**：投资课程理论化，无法与真实市场挂钩

### 本平台的价值主张

用 AI 把"机构级研究流程"开放给每一个大学生，降低认知门槛，培养理性决策习惯

## 痛点问题

### 现有解决方案的不足

**同花顺 / 东方财富等 APP**
- ✅ 数据丰富
- ❌ 只展示数据，不给结论
- ❌ 逻辑不透明，不知道为什么买

**ChatGPT 直接问**
- ✅ 可以对话
- ❌ 没有实时数据接入
- ❌ 幻觉严重，无风控约束

**量化平台（聚宽/BigQuant）**
- ✅ 专业强大
- ❌ 需要编程基础，学习成本极高
- ❌ 面向专业用户，非在校生

### 核心矛盾

```
大学生需要：简单 + 可信 + 有教育性
现有工具给：复杂 OR 不可信 OR 无解释
```

## 研究内容

### 四大模块

**① 实时市场数据层**（对应《智能感知与大数据管理》）
- 覆盖 A股、港股、美股三大市场
- 新浪财经直连接口（< 200ms 响应）
- 财联社 7×24 快讯流

**② 多 Agent 智能分析引擎**（两课共用）
- 4 路专家 Agent 并行分析同一标的
- LLM 驱动，输出结构化报告（Pydantic 强约束）
- 多空辩论机制解决分析师冲突

**③ 风险控制与决策层**（对应《数据驱动的控制与决策》）
- 单股仓位上限 15%（适配大学生小资金场景）
- 止损 5% / 止盈 15% 硬约束
- 最大回撤 15% 熔断，反馈重试最多 2 次

**④ 知识库与教育层**（对应《智能感知与大数据管理》）
- 混合 RAG 检索金融研报
- 财商学长 AI 对话助手
- 模拟撮合引擎（完全纸面交易，不接真实交易所）

## 技术方案

### 整体架构

```
用户输入股票代码
      ↓
  [数据 Agent]  ←── akshare / 新浪财经 / 财联社
      ↓ 并行下发
┌──────────────────────────────────────────┐
│  基本面Agent   技术面Agent   情绪Agent   RAG检索  │
│  PE/PB/ROE    MACD/RSI    新闻情感    研报知识  │
└──────────────────────────────────────────┘
      ↓ 汇聚
  [组合 Agent]  → 发现分歧？
    ├── 是 → [多空辩论 ≤2轮] → 重新决策
    └── 否 → [风控 Agent] → 仓位审核
                ├── 拒绝 → [组合 Agent 修正]  ← 反馈控制
                └── 通过 → [模拟下单] → 输出报告
```

### 核心技术栈

| 层次 | 技术选型 | 对应课程 |
| --- | --- | --- |
| Agent 编排 | **LangGraph** StateGraph 有向图 | 数据驱动控制 |
| 大语言模型 | **阿里云百炼 Qwen3.5-plus** | 两课共用 |
| 多源数据接入 | akshare + 新浪财经 + 财联社 | 智能感知 |
| 向量检索 | **Chroma** + **BM25** 混合 RAG | 大数据管理 |
| 流式传输 | **FastAPI** + SSE 实时事件流 | 智能感知 |
| 数据库 | **SQLite**（可升级 PostgreSQL） | 大数据管理 |

### 数据驱动决策的反馈回路

```
              ┌─────────────────┐
              ↓                 │ 拒绝（带原因）
         风控 Agent  ──────→  组合 Agent 修正
         （硬约束检查）          （调整仓位/方向）
              │
              ↓ 通过
         模拟下单
```

对应课程核心概念：**闭环反馈控制 + 数据驱动修正**

### 大数据感知：多源数据标准化流水线

```
akshare（A股/港股）  ──┐
新浪财经（实时报价）  ──┤→ DataLoader._standardize()
yfinance（美股）     ──┘         ↓
                      [timestamp, open, high, low, close, volume]
                                 ↓
                         技术指标计算（MA/MACD/RSI/KDJ/BOLL）
```

## 风险与可行性

### 技术风险

| 风险 | 严重性 | 应对方案 |
| --- | --- | --- |
| 数据接口变更 | 中 | 多源备用 + 日线数据降级 |
| LLM 幻觉输出 | 高 | Pydantic 强约束 + 置信度阈值过滤 |
| 延迟超时 | 中 | 每节点独立熔断 + 降级为 HOLD |
| API Key 泄露 | 高 | 环境变量管理，不入 git |

### 合规与伦理风险

> ⚠️ **核心红线**：平台绝不接入真实交易所 API

- `TradeOrder.simulated` 恒为 `True`，代码层强制
- 所有输出标注"仅供学习，不构成投资建议"
- 模拟资金 ¥50,000，无法使用真实资金

### 可行性验证

- ✅ 三市场行情接口：< 200ms 响应
- ✅ 全流程 Agent 分析：平均 45-90 秒完成
- ✅ 知识库检索：BM25+Chroma 混合召回率 > 80%
- ✅ 已部署至阿里云服务器，8 张 HTML 页面完整上线

## 我的想法

### 做这个项目学到了什么

**《数据驱动的控制与决策》的收获**
- 反馈控制不只在机器人上——LangGraph 的"风控拒绝 → 修正 → 重试"本质就是离散时间反馈控制
- 多约束决策比单目标优化难得多：仓位、风险、收益之间存在真实的 Pareto 权衡

**《智能感知与大数据管理》的收获**
- 金融数据远比想象中"脏"：akshare 列顺序不一致、新浪财经 GB18030 编码、各接口字段命名混乱
- 大数据管道的瓶颈往往不在计算，而在 I/O——一个同步阻塞调用可以卡死整个服务器

### 如果继续做，我想加入什么

1. **回测引擎**：让用户验证策略历史有效性，而不只是当下推荐
2. **Agent 推理可视化**：可视化每个 Agent 的决策链，让用户理解"为什么买"
3. **流式大数据接入**：接入实时 Tick 数据，升级为真正的流处理架构（Kafka + Flink）
4. **课程联动**：将平台输出与金融课程作业打通，辅助教学评估

### 一点反思

> 做 AI 应用最难的不是模型，而是**数据**和**工程**。

两门课程告诉我"感知数据"和"驱动决策"——这个项目让我明白这两件事有多难同时做好：

**AI 的能力上限 = 数据质量 × 工程健壮性 × 模型能力**

三者缺一不可。理论中的最优控制，在实践里首先要解决的是"数据拿不到"。

## 功能展示 · 个股深度分析

> `analysis.html` — 平台核心功能，LangGraph 全链路实时可视化

### 操作流程

```
① 输入股票代码（如 600519.SH / 00700.HK / AAPL）
② 点击「开始分析」→ SSE 长连接建立
③ 实时观察各 Agent 节点依次点亮
④ 最终生成结构化研报 + 模拟交易建议
```

### 界面核心元素

**状态指示灯**（左上角）
- 🔘 灰色 = 待机
- 🔵 蓝色闪烁 = 分析中
- 🟢 绿色 = 完成
- 🔴 红色 = 异常

**节点进度徽章**（实时流式推送）

| 节点名 | 说明 |
| --- | --- |
| 数据加载 | 抓取行情、财报、新闻 |
| 基本面 | PE/PB/ROE 估值分析 |
| 技术面 | MACD/RSI/KDJ/布林带 |
| 情感面 | 新闻情感建模 |
| RAG 检索 | 金融研报知识召回 |
| 多空辩论 | 分歧时触发，≤ 2 轮 |
| 风控审核 | 仓位 / 回撤硬约束 |
| 指令生成 | 输出模拟交易建议 |

**研报打字机**：分析结论以流式打字机效果实时渲染，包含：评级（BUY / HOLD / SELL）、目标价、置信度、风险提示

### 技术亮点

- SSE 事件流驱动 UI，后端每完成一个节点即推送，无需轮询
- ECharts 渲染财务图表（营收/净利润历史趋势）
- Marked.js 渲染 LLM 输出的 Markdown 研报

## 功能展示 · 模拟交易大厅

> `trade.html` — 三市场模拟撮合，完整账户体系

### 三市场独立账户

| 市场 | 货币 | 初始资金 | 代码示例 |
| --- | --- | --- | --- |
| 🇨🇳 A 股 | CNH 人民币 | ¥100,000 | `600519.SH` |
| 🇭🇰 港 股 | HKD 港币 | HK$100,000 | `00700.HK` |
| 🇺🇸 美 股 | USD 美元 | $10,000 | `AAPL` |

### 下单流程

```
① 左侧行情查询 → 输入代码 → 查询现价 + K 线图
② 自动填入委托面板（代码 + 参考价）
③ 填写委托数量（自动校验最小手数）
④ 估算金额预览 → 提交「模拟委托」
⑤ 撮合引擎执行 → 即时返回成交回报
```

### 账户面板实时显示

- **可用资金** · **持仓市值** · **总资产** · **持仓浮盈** · **累计成交笔数**
- 持仓列表：均价 / 现价 / 浮动盈亏 / 一键快速平仓
- 成交记录：完整历史订单，按市场过滤

### 风控硬约束（代码层强制）

```
单股仓位 ≤ 15%   总仓位 ≤ 50%
止损 -5%         止盈 +15%
TradeOrder.simulated = True  （永远为模拟，不接真实交易所）
```
